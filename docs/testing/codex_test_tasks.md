# Codex Test Tasks

## 목적
Codex가 현재 코드베이스를 읽고, 테스트 공백을 우선순위 있게 찾도록 돕는다.

---

## 요청 원칙

- 핵심 도메인 규칙을 먼저 본다.
- 권한, 멱등성, 상태 전이, 경계값을 우선 본다.
- request spec 우선, system spec은 최소 happy path만 제안한다.
- brittle한 view/html 구조 테스트는 피한다.
- 기존 테스트와 중복되는 제안은 제외한다.

---

## 분석 요청 예시

1. 현재 테스트와 코드 기준으로 핵심 도메인 테스트 공백을 찾아라.
2. coupon/compliment/role 관련 규칙 중 아직 고정되지 않은 부분을 찾아라.
3. request spec으로 가치가 큰 엔드포인트를 우선순위와 함께 제안하라.
4. system spec이 꼭 필요한 부분만 최소 개수로 제안하라.
5. 제안 시 이유, 예상 실패 위험, 추천 spec 종류를 함께 적어라.
