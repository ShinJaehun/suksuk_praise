# 일간·주간·월간 칭찬왕 정책

## 목적

칭찬왕은 특정 시점의 현재 칭찬 집계를 보여 주고 교사가 쿠폰을 편리하게 지급하는 기능이다.
공식 수상 결과를 확정하거나 보존하는 기능은 아니다.

이 문서는 현재 구현과 아직 구현되지 않은 학교 운영일 연동 정책을 구분해 설명한다.

---

## 현재 구현

### 지원 기간과 활성 설정

- 일간(`daily`), 주간(`weekly`), 월간(`monthly`) 칭찬왕을 지원한다.
- 교실마다 각 기간의 칭찬왕 기능을 독립적으로 활성화하거나 비활성화할 수 있다.
- 비활성 기간은 화면과 집계 요청뿐 아니라 서버측 쿠폰 발급에서도 차단한다.
- 수동·custom 쿠폰 발급은 칭찬왕 기간 활성 설정과 무관하다.

### 집계

집계는 `ComplimentKings::Pick`이 담당한다.

- 일간: 해당 날짜
- 주간: 월요일에 시작하는 해당 주
- 월간: 해당 달력 월

기간 계산은 Rails `Time.zone`을 사용한다. 활성 학생만 집계하며, 최고 칭찬 횟수가 같은 학생은 모두 결과에 포함한다.

칭찬왕 결과를 별도 선정 record로 저장하지 않는다. 결과를 조회할 때마다 현재 칭찬 데이터를 다시 집계하므로 같은 기간에도 이후 칭찬 데이터에 따라 결과가 달라질 수 있다.

### 쿠폰 발급과 중복 방지

칭찬왕 쿠폰 발급은 `CouponDraw::Issue`가 처리한다.

- 발급 결과는 `UserCoupon`의 issuance basis, basis tag, period 정보로 구분한다.
- 같은 학생에게 같은 기간의 동일 칭찬왕 쿠폰을 중복 발급하지 않는다.
- 이미 사용 처리된 쿠폰도 같은 기간에는 다시 발급하지 않는다.
- 비활성화된 기간의 칭찬왕 쿠폰은 실제 쿠폰 생성 전에 거부한다.
- 쿠폰 발급 후 칭찬 순위가 바뀌더라도 기존 발급을 자동으로 취소하거나 보정하지 않는다.

---

## 확정된 향후 학교 운영일 정책

다음 정책은 확정되었지만 학교 휴무일과 운영일 계산은 아직 구현되지 않았다. 현재 시스템에는 마지막 학교 운영일에 따른 사용 날짜 제한이 없다.

- 주간 칭찬왕은 해당 주의 마지막 학교 운영일에만 사용할 수 있도록 한다.
- 월간 칭찬왕은 해당 달의 마지막 학교 운영일에만 사용할 수 있도록 한다.
- 사용 가능한 날의 00:00부터 그날이 끝날 때까지 결과를 여러 번 열고 닫고 다시 조회할 수 있다.
- 주간·월간 칭찬왕은 운영일 연동 후에도 공식 수상 결과를 확정·보존하지 않는다.
- 쿠폰 발급 후 순위가 바뀌어도 시스템은 자동 보정하지 않는다.
- 추가 지급이 필요하면 교사가 custom 쿠폰으로 처리한다.
- 사용 가능 날짜를 놓쳐도 소급 선정하거나 미선정 상태를 관리하지 않는다.
- 별도의 칭찬왕 확정 모델을 만들지 않는다.

학교 휴무일, 공식 공휴일 동기화, 마지막 학교 운영일 계산의 상세 설계는 학교 일정 기능에서 다룬다.
